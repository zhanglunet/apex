# APEX V1.2 Evidence Layer Release Notes

## 版本目标

V1.2 Evidence Layer 的目标，是把 R1 Meeting Intelligence 从“结构化会议纪要”推进到“可审计研究对象”。

V1.1 已经具备质量闭环：

```text
生成 -> 修订 -> Failure Card -> Eval Case -> Memory Object -> smoke test
```

V1.2 在此基础上新增证据层：

```text
R1 输出 -> Evidence Items -> Evidence Panel -> Evidence Workbench -> Failure Card
```

## 产品能力

### Evidence Item

每次 R1 生成后，系统会从结构化输出中提取 Evidence Items：

- `keyChanges[].evidence`
- `actionItems[].evidence`
- `openQuestions[]`
- `qualityWarnings[]`

Evidence Item 包含：

- section
- claim
- evidenceText
- sourceHint
- status：`SUPPORTED` / `WEAK` / `MISSING`

### Run Detail Evidence Panel

单个 Run 内可以查看：

- SUPPORTED 数量。
- WEAK 数量。
- MISSING 数量。
- 每条证据的 claim、evidenceText、section 和 sourceHint。

### Evidence Workbench

`/evidence` 页面支持：

- 查看全部 Evidence Items。
- 按 status 筛选。
- 按 section 筛选。
- 跳转回来源 Run。
- 对 WEAK / MISSING 证据项创建 Failure Card。

### Dashboard Evidence Metrics

Dashboard 新增：

- Missing Evidence。
- Weak Evidence。
- Supported Evidence。

这些指标直接链接到 `/evidence` 的对应筛选视图。

## 工程验证

本轮每个小版本均执行：

```bash
npm run build
APP_URL=http://localhost:3001 npm run test:smoke
```

smoke test 当前覆盖：

- `/api/health`
- 上传文件
- 创建 Run
- 生成 R1 输出
- Evidence Item 写入
- Evidence 缺口创建 Failure Card
- 保存修订
- Failure Card 创建与校验
- Eval Case 创建与幂等保护
- Markdown 导出
- Dashboard / Inbox / Runs / Evidence / Failure Ops / Evals / Memory 页面

## 当前限制

V1.2 仍然不做：

- 原文逐字定位。
- PDF 证据引用。
- 向量检索。
- 多用户权限。
- 外部系统集成。

这些能力留给后续 Evidence 深化或 Research Desk 阶段。

## 下一步建议

建议下一个阶段进入 `V1.3 Evidence Deepening`：

- 让 Evidence Item 支持人工状态更新。
- 给 Evidence Item 增加 source snippet 或手动 source hint 编辑。
- 让 Eval Case 的 expectedBehavior 更明确引用 Evidence claim。
- 增加 Evidence completeness score。
